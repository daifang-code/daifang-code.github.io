主机字节顺序与网络字节顺序的错误转换

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述
  • 漏洞与风险
  • 漏洞利用威胁
  • 缓解与预防
  • 演示实例
    • 漏洞代码示例
    • 漏洞修复代码示例
  • 引用

缺陷描述

来源于主机边界的多字节的值在发送或者写入到环境之前，未将主机字节顺序转换为网路字节顺序或从外界获取或者读取一个多字节到主机边界时，没有把网络字节顺序转换为主机字节顺序。

漏洞与风险

程序在系统下运行伴随不同的字节顺序方法，这些方法需相互之间连通。错误地转换字节顺序可能导致程序功能实现不正确。

漏洞利用威胁

低

缓解与预防

应使用htons函数把unsigned short类型从主机序转换到网络序；应使用htonl 函数把unsigned long类型从主机序转换到网络序；应使用ntohs 函数把unsigned short类型从网络序转换到主机序；应使用ntohl 函数把unsigned long类型从网络序转换到主机序。

演示实例

漏洞代码示例

1
2
3
4
void example_fun(int a, long b) 
{
    send(a, &b, sizeof b, 0);   // b传递时没有被转换为网络字节顺序
}

漏洞修复代码示例

1
2
3
4
5
void example_fun(int a, long b) 
{
    int c = htonl(b);
    send(a, &c, sizeof c, 0); 
}

不正确的移位运算

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述
  • 漏洞与风险
    • 未定义的行为
  • 漏洞利用威胁
  • 缓解与预防
  • 演示实例
    • 漏洞代码示例
    • 漏洞代码修复示例

缺陷描述

移位操作包含"移位表达式<<加减表达式"形式的左移位操作和"移位表达式>>加减表达式"形式的右移位操作。每个整型的操作数都是执行整型提升。结果的类型是经过提升的左操作数的类型。如果右操作数是负值或大于等于提升后左操作数的位数，其行为是未定义的。

移位的数量不能是负数或大于等于提升后左操作数的位数。一个整型的精度是它表示数值的位数，包含符号位和填充位。对于无符号整型来说，它的宽度和精度是一样的。然而对于有符号整数来说，它的宽度大于它的精度。这条规则使用精度代替宽度来表示，因为几乎在每一种情况下，在移位操作中移位的数量大于或等于操作数的精度时会导致一个错误（逻辑错误）。不同于溢出，逻辑错误是一种表示形式方面的缺陷。

漏洞与风险

未定义的行为

移位操作包含"移位表达式<<加减表达式"形式的左移位操作和"移位表达式>>加减表达式"形式的右移位操作。每个整型的操作数都是执行整型提升。结果的类型是经过提升的左操作数的类型。如果右操作数是负值或大于等于提升后左操作数的位数，其行为是未定义的。

漏洞利用威胁

中

缓解与预防

保证移位操作的右操作数为正数并且小于左操作数的位数。

演示实例

漏洞代码示例

long long bad(long long value, int shift_size)
{
    if (shift_size < 64) 
      return value;

    return (value << shift_size);
}

在上面的代码中，移位操作 value << shift_size 的右操作数大于等于64，而左操作数的位数为64，导致函数返回一个未定义的结果。

{

      return val;

    return (value << 32);
}

在上面的代码中，移位操作 value << 32 的右操作数为32，小于左操作数的位数64，是一个合法的操作。

整数溢出

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述

  • 漏洞利用威胁
  • 缓解与预防
  • 演示实例
    • 漏洞代码示例
    • 漏洞代码修复示例1

缺陷描述

当表达式的结果超出结果类型值范围时表达式结果会与真实算数结果有差别，如果程序需要使用真实的算数结果做后续操作会导致操作结果与期待不符，比如加上一个明确为正数的值确比原始值还小，或者两个无符号非零整数相加等于0

漏洞与风险

操作结果与期待不符

漏洞利用威胁

高危（如果期待算术操作结果，例如计算大小，计算资源消耗、金钱交易等） 无威胁（如果期待计算机逻辑操作结果，比如算hash，算加密等）

缓解与预防

使用前检查操作数取值范围 使用更大区间的数据类型

演示实例

漏洞代码示例

unsigned int x = 4000000000;
unsigned int y = 4000000000;
bool test() {
    return (x + y > x); // Return false
}

在上面的代码中，x+y超越了32位无符号int类型的上限4294967295，最高位被丢失，导致相加的结果反而更小

漏洞代码修复示例1

unsigned long long x = 4000000000;
unsigned long long y = 4000000000;
bool test() {
    return (x + y > x); // Return true
}

在上面的代码中，使用64位的变量类型，避免整数溢出。

不精确的数学表达式

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述
  • 漏洞与风险
    • 数值精度
  • 缓解与预防
  • 漏洞利用威胁

缺陷描述

当对类型进行强制转换时，由于每种类型所占的字节数不同，这就会导致计算结果的丢失。

漏洞与风险

数值精度

浮点型转换成整型就会造成精度损失，在数值计算中，要非常小心，对精度非常有要求。基类指针转换成派生类指针风险非常大。

缓解与预防

以前类型转换，就直接用int(a)，double(b)等方式。这种方式不区分类型转换的风险，并且有些bug是类型转换引起的，如果显性的来进行强制类型转换，可以方便bug排查。比如基类和派生类的转换风险很大，用B*(A)就非常不合适，但用关键字进行强制类型转换，就可以用查找的方式在程序中找到不同风险级别的转换。

漏洞利用威胁

低

演示示例

漏洞缺陷示例

void bad() {
  float a=10.01;
  int b;
  b=a;//detect
}

漏洞缺陷修复示例

  float a=10.01;
  int b;
  float(b)=a;
}

将a赋值给转换为浮点数的b。

类型提升前整数溢出

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述
  • 漏洞与风险
    • 整数溢出
  • 漏洞利用威胁
  • 缓解与预防
  • 演示实例
    • 漏洞代码示例
    • 漏洞代码修复示例

缺陷描述

两个整数在进行算术运算时，字节数小的类型会提升到字节数大的类型，然后进行运算。如果计算的结果超出了字节数大的类型的范围，会导致溢出。 例如，在两个整数的乘法运算中，结果可能会超出整数的表示范围。

漏洞与风险

整数溢出

整数溢出导致结果超出开发人员的预期，导致未知行为的发生。

漏洞利用威胁

中

缓解与预防

为了避免整数溢出，在算术运算之前将任意操作数提升为字节数大的类型。

演示实例

漏洞代码示例

1
2
3
4
5
6
7
8
void bad() {
  unsigned int x = 2147483648;
  unsigned int y = 2;
  unsigned long long z;
  if ((x * y) == z) { // detect
    // Do something.
  }
}

在上面的代码中，x * y 的结果超出了unsigned int的最大范围(2^32 − 1 即 4,294,967,295)。

漏洞代码修复示例

1
2
3
4
5
6
7
8
void good() {
  unsigned int x = 2147483648;
  unsigned int y = 2;
  unsigned long long z;
  if (((unsigned long long)x * y) == z) { 
    // Do something.
  }
}

在上面的代码中，为了避免 x * y 导致的整数溢出，将x提前转换成 unsigned long long 类型。

char数据类型未指定是否有符号

缺陷描述

定义char数据类型的变量时未指定是否有符号。

漏洞与风险

在C语言标准中没有明确定义char数据类型是否有符号，一些编译器允许使用编译器开关切换char类型是否有符号，可能导致代码难以移植。

漏洞利用威胁

低

缓解与预防

注明char数据类型是否有符号，或使用typedef或#define定义的数据类型。

演示实例

漏洞代码示例

1
static char *s = "Hello, welcome!\n";  //char数据类型未指定是否有符号

漏洞修复代码示例

1
2
typedef unsigned char UCHAR;
static UCHAR *s = "Hello, welcome!\n";

浮点数的相等或不等比较

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

缺陷描述

两个浮点型或双精度浮点型的值使用"=="或"!="运算符进行比较。

漏洞与风险

浮点数在计算机中的存储和计算是不精确的，因此对浮点数采用"=="或"!="运算符进行比较可能得到与期望结果相反的结果。

漏洞利用威胁

低

缓解与预防

通过计算两个浮点数之间的差的绝对值是否小于一个允许的误差值判断浮点数之间是否相等或不等。

演示实例

漏洞代码示例

{

    return true;

}

除零错误

在 2024-九月-28 发布于 数值计算错误 分类 • 1 min read

  • 缺陷描述

缺陷描述

对整数执行的除法或余数运算时易受到零除错误的影响。因此，在对整数类型进行除法或余数运算时，除数必须在运算之前检查为零。对浮点数执行的除法和余数运算不受此规则约束。

漏洞与风险

程序崩溃

在程序执行除法运算时，如果除数为零，会发生运行时错误。通常会导致程序异常终止。

漏洞利用威胁

高

缓解与预防

在执行除法运算前，尤其是将运算结果等不确定值作为除数的时候，一定要对除数进行非零验证。

演示实例

漏洞代码示例 1

1
2
3

  int b = 0;
    return a / b;
}

在上面的代码中，在除法运算a/b时，除数b为0，导致程序崩溃。

漏洞代码修复示例 1

1

  int b = 0;
    if(b != 0)
      return a / b;
    return a;
}

在上面的代码中，在执行除法运算a/b之前，验证了除数b不为0。
