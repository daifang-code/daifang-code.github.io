PE0016: 多余的分号

在if，for，while语句后面误用多余的分号，改变代码的逻辑。

缺陷信息

缺陷ID
PE0016
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

当一个if语句被分号提前终止时，then语句将永远被执行，从而改变了if语句的逻辑。在while,for等循环语句被分号提前终止时，可能会导致死循环或者循环体部分只被执行一次。

漏洞与风险

逻辑错误

当一个if语句被分号提前终止时，then语句将永远被执行，从而改变了if语句的逻辑。在while,for等循环语句被分号提前终止时，可能会导致死循环或者循环体部分只被执行一次。

漏洞利用威胁

中

缓解与预防

仔细检查if，for，while语句后面的逗号，移除多余的分号。

演示实例

漏洞代码示例
void bad() {
    while (1){};
    
  while (1); // detect
  {
    if (1)
      return;
  }
}

在上面的代码中，在第4行while语句后面存在一个多余的分号，导致死循环。

漏洞代码修复示例
void good() {
    while (1){};
    
  while (1)
  {
    if (1)
      return;
  }
}

在上面的代码中，在第4行移除while语句后面的分号，使得程序逻辑正常执行。
