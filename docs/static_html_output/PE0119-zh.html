PE0119: 使用缺乏一致性定义的long long类型

使用缺乏一致性定义的long long类型。

缺陷信息

缺陷ID
PE0119
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

使用在跨编译器上缺乏一致性定义的数据类型，如"long long"类型。

漏洞与风险

使用在跨编译器上缺乏一致性定义的数据类型可能导致代码难以移植。

漏洞利用威胁

低

缓解与预防

应使用编译器内置的或定义的宏来表示实际想要的大小，而不是假设编译器提供的类型有一个合理的大小。

演示实例

漏洞代码示例
void example_fun()
{
    long long a;  //使用缺乏一致性定义的long long类型
}
漏洞代码修复示例
void example_fun()
{
    UINT64_t a;
}
