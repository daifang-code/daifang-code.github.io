缺陷描述

考虑返回一个零长度数组而不是null

漏洞与风险

返回长度为零的数组而不是空引用来表示没有结果通常是更好的设计（即，结果的空列表）。 这样，该方法的客户端不需要显式检查null。

另一方面，使用null表示“这个问题没有答案”可能是合适的。 例如，如果给定一个不包含文件的目录，File.listFiles() 将返回一个空列表，如果该文件不是目录，则返回null。
