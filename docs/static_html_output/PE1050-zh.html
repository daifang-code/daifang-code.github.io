PE1050: 无效函数参数

形参与实参类型不一致。

缺陷信息

缺陷ID
PE1050
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

函数传递的实参类型与形参不一致

漏洞与风险

如果实参和形参不一致，则最终调用以形参为准。这样会在调用函数时改变参数的类型，从而导致结果的差异。

缓解与预防

使形参和实参类型一致，如果不能一致则在编写时考虑形参的类型。

漏洞利用威胁

高

演示示例

漏洞缺陷示例
void bad(int a);
char b=30; //detect
bad(b)

b是实参，它与形参的类型是不一样的。在调用时系统将默认b为int型并带入函数中。但是这样会导致b类型的改变

漏洞缺陷修复示例
void bad(int a);
int b=30;
bad(b)
