PE1052: Imprecise math expression

Bug info

BugTypeID
PE1052
Tags
C, CPP
Category
Checkers/CPP
Authors
pinpoint

Description

When the type is cast, the number of bytes occupied by each type is different, which results in the loss of the calculated result.

Vulnerability and risk

Numerical Precision

The conversion of float type to integer type will result in loss of precision.The conversion from base class pointers to derived class pointers is risky.

Likelihood of Exploit

Previous type conversions used int(a), double(b), and so on. This method does not distinguish the risk of type conversion, and some bugs are caused by type conversions. If we do a type conversion explicitly, we can facilitate bug troubleshooting. For example, the conversion of the base class and derived class is very risky, and it is not appropriate to use B*(A), but using the keyword to perform the forced type conversion, you can find the conversion of different risk levels in the program using the search method.

Potential Mitigations

medium

Demonstrative Examples

Vulnerable Code Example
void bad() {
  float a=10.01;
  int b;
  b=a;//detect
Fixed Code Example
void good() {
  float a=10.01;
  int b;
  float(b)=a;
}

Assigning a to b that the type from int to float.
