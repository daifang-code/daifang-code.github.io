PE4025: Incompatible types specifier compiler error

Incompatible types specifier compiler error.

Bug info

PE4025
Tags
C, CPP

Checkers/CPP
Authors
pinpoint

Description

As language standards continue to update, compilers are constantly updating to support new one. While language standard updates provide new features, they may also be incompatible with previous versions. This creates a compiler incompatibility issue, code that uses the new version of the language standard may not work properly in compilers that support older versions of the standard.

Vulnerability and risk

Incompatibility with the compiler may make the software not work properly or even compile.

Likelihood of Exploit

Low

Potential Mitigations

Write code according to the language standard supported by the compiler.

Demonstrative Examples

Vulnerable code Example
auto r_it = r.begin();
auto r_end = r.end();

auto is used in C++98 to declare variables as automatic variables and has an automatic lifetime. For example, auto int a = 1; declare the integer a as a automatic variable in C++98. In C++11, auto can automatically select the matching type for this variable based on the type of the initial value of the variable when declaring the variable. This problem code has compatibility issues in the C++98 compiler.
