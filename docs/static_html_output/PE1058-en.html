Description

If the class member function is a const function, the function can be technically satisfied, and this function does not show compilation errors, but it is conceptually meaningless. Class member functions can be static. This will bring performance benefits because the instance is not passed to the function. This change does not cause a compiler error, but it is not necessarily significant conceptually .

Likelihood of Exploit

Consider the design and function tasks first. Is it a function that cannot change the internal state of an object? If not, try not to use const and static functions.
