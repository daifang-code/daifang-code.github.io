PE1024: 初始化成员的顺序不正确

初始化列表成员顺序不正确可能导致部分成员未正确初始化。

缺陷信息

缺陷ID
PE1024
标签
CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

初始化列表成员顺序不正确可能导致部分成员未正确初始化。

漏洞与风险

未正确初始化

成员变量的初始化顺序由内存中的排列顺序决定。如果对象A的初始化依赖于对象B，而对象A比对象B先初始化，这回导致初始化不正确。

漏洞利用威胁

低

缓解与预防

保证正确的初始化顺序。

演示实例

漏洞代码示例
class A
{
  private:
    int a,b;
  public:
    A():b(1),a(b+1){}
    ...
}

在上面的代码中，初始化的顺序为a,b，但a的初始化依赖于b。

漏洞代码修复示例
class A
{
  private:
    int b,a;
  public:
    A():b(1),a(b+1){}
    ...
}

在上面的代码中，保证了正确的初始化顺序。
