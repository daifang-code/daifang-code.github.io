缺陷描述

NullCipher不安全

漏洞与风险

NullCipher很少在正式的应用程序中使用。它通过返回与提供的明文相同的密文来实现Cipher接口。在一些情况下，例如测试，可以使用NullCipher。

易受攻击的代码：

Cipher doNothingCihper = new NullCipher();
[...]
//产生的密文与明文相同。
byte[] cipherText = c.doFinal(plainText);

解决方案：
避免使用NullCipher。 其意外使用可能会带来严重的安全风险。

漏洞利用威胁

严重
