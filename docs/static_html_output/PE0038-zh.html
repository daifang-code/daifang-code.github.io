PE0038: 死代码

从来不被执行的代码称为死代码。

缺陷信息

缺陷描述

从来不被执行的代码称为死代码。死代码的存在提示程序中可能存在逻辑错误，可能导致错误的程序行为。

漏洞与风险

逻辑错误

从来不被执行的代码称为死代码。死代码的存在提示程序中可能存在逻辑错误，可能导致错误的程序行为。

漏洞利用威胁

低

缓解与预防

程序员确定为什么这段代码永远不会执行，并正确解决这个问题。

演示实例

漏洞代码示例
void bad()
{
  String s = null;
  if (b) {
    s = "Yes";
    return;
  }

  if (s != null) {
    Dead();
}

在上面的代码中，第二个if条件判断永远为false，因为s的值在这个地方永远为null。

漏洞代码修复示例
void good()
{
  String s = null;
  if (b) {
    s = "Yes";
    return;
  }
}

在上面的代码中，移除死代码。
