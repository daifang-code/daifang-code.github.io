缺陷描述

方法调用低效的Number构造函数，应改为使用静态的valueOf

漏洞与风险

使用new Integer(int)方法总是会创建一个新的对象，而Integer.valueOf(int)允许由编译器，类库或JVM完成值的缓存。使用缓存值的方式来避免对象的分配可以获得更好的代码性能。

如果保证-128和127之间的值具有相应的高速缓存实例，那么使用<code>valueOf</code>比使用构造函数快大约3.5倍。 对于超出恒定范围的值，两种方法的性能是相同的。

除非类必须符合Java 1.5以前的JVM，否则请使用autoboxing或valueOf()方法创建Long, Integer, Short, Character, and Byte实例。
