缺陷描述

用零个线程创建ScheduledThreadPoolExecutor

漏洞与风险

(Javadoc) 用零个线程创建的ScheduledThreadPoolExecutor永远不会执行任何操作。后续更改线程池大小也无济于事。因为ScheduledThreadPoolExecutor在执行中修改线程数是无效的，在创建的时候设置线程数量将会一直保持。
