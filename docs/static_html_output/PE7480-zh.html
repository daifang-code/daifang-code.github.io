缺陷描述

不可信的servlet参数

漏洞与风险

Servlet可以从各种方法读取GET和POST参数。 所获得的价值应该被认为是不安全的。 在将这些值传递给敏感的API之前，您可能需要验证或清除这些值，例如：

  • SQL查询（可能导致SQL注入）
  • 文件打开（可能导致路径遍历）
  • 命令执行（潜在的命令注入）
  • HTML结构(潜在的XSS)
  • ......
