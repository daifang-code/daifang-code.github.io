缺陷描述

三元运算符对基础类型（primitive）值进行拆箱和强制类型转换

漏洞与风险

在三元运算符操作时对值进行拆箱并转换为另一种基础类型。例如： b ? e1 : e2。Java的语义要求如果e1和e2是包装的数值，则将值取消装箱并转换/强制转换为它们的公共类型（例如，如果e1的类型为Integer，而e2的类型为Float，则e1将被取消装箱，转换为一个浮点值，并装箱。参见JLS第15.25节。
