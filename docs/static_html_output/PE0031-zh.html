PE0031: 在构造函数中未初始化所有非静态的成员变量

在构造函数中未初始化所有非静态的成员变量。

缺陷信息

缺陷ID
PE0031
标签
CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

一个类的构造函数通常需要初始化类中所有的非静态成员变量。这是一种常见的代码标准。未初始化成员变量是一种不安全的行为，因为使用未初始化的变量会导致意料之外的情况发生。

漏洞与风险

意料之外的行为

由于分配给变量的内存可能在之前某个程序或函数使用过，因此使用未初始化的变量可能产生意想不到的异常行为和错误结果。

漏洞利用威胁

中

缓解与预防

在构造函数中初始化所有非静态的成员变量。

演示实例

漏洞代码示例
class Bad {
public:
  int i;
  char c;
  float f;
  static d;
  Bad():f(1.0){}	// 'i' and 'c' are uninitialized!
};

在上面的代码中，在Bad的构造函数中，'i'和'c'未被初始化。

漏洞代码修复示例
class Good {
public:
  int i;
  char c;
  float f;
  static d;
  Good():i(0), c('a'), f(1.0){}	// all of the members are initialized!
};

在上面的代码中，在Good的构造函数中初始化所有的非静态成员变量。
