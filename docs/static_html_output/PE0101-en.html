PE0101: Implicit conversions between built-in types and booleans

Description

This check can be used to find implicit conversions between built-in types and booleans. Depending on use case, it may simply help with readability of the code, or in some cases, point to potential bugs which remain unnoticed due to implicit conversions.

Vulnerability and risk

A variable of type boolean only can take one of two values, true or false. Adding or subtracting a variable of type boolean is meaningless, which may result in incorrect implementation of the program function.

Likelihood of Exploit

Low

Potential Mitigations

The boolean type and the built-in type are not implicitly converted to each other.

Demonstrative Examples

Vulnerable Code Example
void example_fun(bool a)
{
  a = a + 2;  // Addition of boolean type variables
}
