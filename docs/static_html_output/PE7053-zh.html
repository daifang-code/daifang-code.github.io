缺陷描述

clone方法没有调用super.clone()

漏洞与风险

非final类定义了一个不调用super.clone()的clone()方法。 如果此类（“A”）由子类（“B”）扩展，并且子类B调用super.clone()，那么B的clone()方法可能会返回一个 A 类型的对象，这违反了clone()的标准合约。 如果所有clone()方法都调用super.clone()，那它们将保证使用Object.clone()返回正确类型的对象。
