PE22422: E714

Test for object identity should be 'is not'

Bug info

BugTypeID
PE22422
Tags
flake8
Category
Code style and efficiency
Authors
pinpoint

Tests for object identity should use the form x is not None rather than not x is None. The former example is simply more readable.

Anti-pattern
if not user is None:
    print(user.name)
Best practice
if user is not None:
    print(user.name)
