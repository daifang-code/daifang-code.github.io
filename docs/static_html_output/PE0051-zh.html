PE0051: 实参顺序与形参顺序不一致

调用函数时，传递的实参与形参顺序不一致

缺陷信息

缺陷ID
PE0051
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

当调用函数时，传递的实参与形参顺序不一致会导致程序功能实现不正确。

漏洞与风险

意料之外的结果

当调用函数时，传递的实参与形参顺序不一致会导致程序功能实现不正确。

漏洞利用威胁

低

缓解与预防

调用函数时，传递的实参与形参顺序保持一致。

演示实例

漏洞代码示例
bool authenticate(char* username, char* password) { 
  // 验证用户名和密码的正确性
}

void bad(char* username,char* password){
  authenticate(password, username);  //传递的实参与形参顺序不一致。
}

在上面的代码中，调用函数authenticate时，传递的实参与形参顺序不一致。

漏洞代码修复示例
bool authenticate(char* username, char* password) { 
  // 验证用户名和密码的正确性
}

void good(char* username, char* password){
  authenticate(username, password); 
}

在上面的代码中，调用函数authenticate时，传递的实参与形参顺序一致。
