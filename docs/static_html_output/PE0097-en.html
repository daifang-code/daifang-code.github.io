PE0097: Suspicious array to pointer conversion

Suspicious array to pointer conversion.

Bug info

BugTypeID
PE0097
Tags
C, CPP
Category
Checkers/CPP
Authors
pinpoint

Description

Use array names to access elements in an array. Array names are converted to pointers. Usually caused error by missing index data.

Vulnerability and risk

Accessing data which is missing index can result in incorrect results.

Likelihood of Exploit

Low

Potential Mitigations

The index number should be used to access the array elements.

Demonstrative Examples

Vulnerable Code Example
struct S
{
  char c; 
};
void example_fun()
{
  struct S a[5];
  for (int i = 0; i < 5; i++)
  {
    a->c = 'a'; //The array name is converted to a pointer, and the   
                //first element is always accessed.
  }
}

In the above code, the array name is converted to a pointer, and the first element is always accessed.

Fixed Code Example
struct S
{
  char c; 
};
void example_fun()
{
  struct S a[5];
  for (int i = 0; i < 5; i++)
  {
    a[i]->c = 'a'; 
  }
}

In the above code, the index number is used to access the array elements.
