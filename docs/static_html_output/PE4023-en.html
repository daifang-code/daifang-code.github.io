PE4023: Unintended integer division

Unintended integer division

Bug info

BugTypeID
PE4023
Tags
C, CPP
Category
Checkers/CPP
Authors
pinpoint

Description

In many programming languages, the result of dividing two integers automatically rounds to the nearest integer towards zero. Such an division is suspicious if it is performed in a context of floating point arithmetic.

Likelihood of Exploit

Critical

Demonstrative Examples

double x=22/7;

double test_tp1() {
    return x;
}

double test_tp2() {
    return 22/7;
}

double test_tp3() {
    return double(22/7);
}

double test_tp4() {
    return double(22/f());
}

double test_fp1() {
    return double(22/2);
}
