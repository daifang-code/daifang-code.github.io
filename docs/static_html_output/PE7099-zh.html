缺陷描述

使用Random的nextInt方法而不是nextDouble来生成随机整数

漏洞与风险

如果r是java.util.Random，则可以使用r.nextInt(n)生成从0到n-1的随机数，而不是使用(int)(r.nextDouble() * n)。 -r.nextInt(100)产生的随机数都是非负数。例如，如果要生成从-99到0的随机值，请使用-r.nextInt(100)。
