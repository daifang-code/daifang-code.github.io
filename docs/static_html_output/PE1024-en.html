PE1024: Fields are initialized unorderedly

Member variable is in the wrong place in the initializer list. Members are initialized in the order they are declared, not in the order they are in the initializer list.

Description

Member variable is in the wrong place in the initializer list. Members are initialized in the order they are declared, not in the order they are in the initializer list.

Vulnerability and risk

Wrong in initialization

Member variable is in the wrong place in the initializer list. Members are initialized in the order they are declared, not in the order they are in the initializer list.

Likelihood of Exploit

low

Potential Mitigations

Keeping the initializer list in the same order that the members were declared prevents order dependent initialization errors.

Demonstrative Examples

bad Example
class A
{
  private:
    int a,b;
  public:
    A():b(1),a(b+1){}
    ...
}

In this code, the initialization sequence is a, b. But the initialization of a depends on b.

good Example
class A
{
  private:
    int b,a;
  public:
    A():b(1),a(b+1){}
    ...
}

In this code, ensure the correct initialization order.
