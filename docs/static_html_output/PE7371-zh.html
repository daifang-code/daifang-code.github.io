缺陷描述

线程实例上调用的静态Thread.interrupted()方法

漏洞与风险

此方法在Thread对象上调用Thread.interrupted()方法，该对象看起来不是当前线程的Thread对象。由于interrupted()方法是静态的，因此将在与程序员想要的对象不同的对象上，调用中断的方法。

漏洞利用威胁

低
