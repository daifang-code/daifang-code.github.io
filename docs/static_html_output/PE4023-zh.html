PE4023: 可疑的整数除法

可疑的整数除法

缺陷信息

缺陷ID
PE4023
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

使用整数除法（可能打算使用浮点除法）。 根据语言规则，将两个整数类型的值相除（long，unsigned等）会将商计算为整数（整数除法），实际上是四舍五入到零或忽略任何余数。当整数除法发生在需要浮点值的上下文中时，它是可疑的，因为程序员可能希望代表非整数商。当检查器发现此模式时，它会报告缺陷，同时至少还有一个其他指示可能是分数商的指示。

漏洞利用威胁

严重

演示实例

漏洞代码示例
double x=22/7;

double test_tp1() {
    return x;
}

double test_tp2() {
    return 22/7;
}

double test_tp3() {
    return double(22/7);
}

double test_tp4() {
    return double(22/f());
}

double test_fp1() {
    return double(22/2);
}
