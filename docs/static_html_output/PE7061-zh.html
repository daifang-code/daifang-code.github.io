缺陷描述

徒劳地尝试更改ScheduledThreadPoolExecutor的线程池大小

漏洞与风险

(Javadoc)

ScheduledThreadPoolExecutor在执行中修改线程数是无效的，在创建的时候设置线程数量将会一直保持。

虽然ScheduledThreadPoolExecutor继承自ThreadPoolExecutor，但继承的一些调优方法对它没用。ScheduledThreadPoolExecutor使用corePoolSize线程和无界队列固定线程池的大小，所以对maximumPoolSize的调整没有任何效果。 </p>
