PE0105: 在调用函数时对传递的参数进行赋值

在调用函数时对传递的参数进行赋值。

缺陷信息

缺陷ID
PE0105
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

在调用函数时对传递的参数进行赋值。

漏洞与风险

在C标准中求值顺序没有定义，所以在不同的编译器上移植时使用语法捷径可能产生操作影响。这种赋值并不总是一定会发生在函数调用之前，在不同的平台上将产生不同的结果。

漏洞利用威胁

低

缓解与预防

不应在调用函数时对传递的参数进行赋值。

演示实例

漏洞代码示例
void example_fun1(int a)
{
    ...
}
void example_fun2(int a)
{
    example_fun1(a = a + 1);  //在调用函数时对传递的参数进行赋值
    ...
}
漏洞修复代码示例
void example_fun1(int a)
{
    ...
}
void example_fun2(int a)
{
    a = a + 1
    example_fun1(a); 
    ...
}
