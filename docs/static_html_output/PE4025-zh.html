缺陷描述

随着语言标准的不断更新，编译器也在不断更新支持新的标准。语言标准更新提供新特性的同时，可能也会与之前的版本不兼容。这就造成了编译器不兼容的问题，使用新版本语言标准的代码未必能在支持旧版本标准的编译器中正常运行。

漏洞与风险

编译器与代码不兼容可能使得软件无法正常运行，甚至无法通过编译。

漏洞利用威胁

低

缓解与预防

根据编译器支持的语言标准编写代码。

演示实例

漏洞代码示例
auto r_it = r.begin();
auto r_end = r.end();

auto在C++98中，用于声明变量为自动变量，拥有自动的生命期。例如，在C++98中auto int a = 1;声明整型变量a是一个自动变量。 而在C++11中，auto可以在声明变量的时候根据变量初始值的类型自动为此变量选择匹配的类型。 这段问题代码在C++98编译器存在兼容性问题。
