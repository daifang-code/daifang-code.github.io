PE0011: 枚举值用作布尔值

枚举值被误用作布尔值。

缺陷信息

缺陷ID
PE0011
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

在某些情况下，程序员将枚举值误用作布尔值作为条件判断，使得条件恒为真或恒为假，结果可能与程序员的意愿不符，导致未定义的行为。

漏洞与风险

未定义的行为

将枚举值误用作布尔值作为条件判断，可能会导致未定义的行为发生。

漏洞利用威胁

中

缓解与预防

不要将枚举值用作布尔值进行条件判断。

演示实例

漏洞代码示例
enum color { red, green, blue };

void bad() {
  color c = green;

  if (c) // detect
  {
    c = red;
  }
}

在上面的代码中，在第6行中使用枚举值c作为if的条件判断，结果恒为真。

漏洞代码修复示例
enum color { red, green, blue };

void good() {
  color c = green;

  if (c == green) 
  {
    c = red;
  }
}

在上面的代码中，在第6行中使用布尔表达式作为if的条件判断。
