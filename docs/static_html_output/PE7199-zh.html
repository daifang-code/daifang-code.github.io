缺陷描述

方法忽略InputStream.read()的结果

漏洞与风险

这个方法忽略了其中一个可以返回多个字节的java.io.InputStream.read()变体的返回值。 如果未检查返回值，则调用者将无法正确处理读取的字节数少于调用者请求的字节数的情况。 这是一种特别隐蔽的错误，因为在许多程序中，从输入流中读取通常会读取所请求的全部数据，导致程序运行失败。

漏洞利用威胁

低
