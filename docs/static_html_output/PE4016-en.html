PE4016: Non optimal padding function

The optimal asymmetric cryptographic padding is not used when using the RSA encryption algorithm.

Bug info

BugTypeID
PE4016
Tags
C, CPP
Category
Checkers/CPP
Authors
pinpoint

Description

The optimal asymmetric cryptographic padding is not used when using the RSA encryption algorithm.

Vulnerability and risk

Failure to use optimal asymmetric cryptographic padding will reduce the difficulty of attacker decryption.

Likelihood of Exploit

Medium

Potential Mitigations

Use the asymmetric asymmetric encryption padding when using the RSA encryption algorithm.

Demonstrative Examples

Vulnerable Code Example
#include "rsa.h"
using namespace CryptoPP;
int encryptor(char* input, char* output)  //Encryption function, the output parameter returns the encrypted string
{
    RSAES_PKCS1v15_Encryptor rsapkcs1;  //Fill with Pkcs1
    int r = rsapkcs1.enc(input, output);
    ...
}
Fixed code example
#include "rsa.h"
using namespace CryptoPP;
int encryptor(char* input, char* output)  //Encryption function, the output parameter returns the encrypted string
{
    RSAES_OAEP_SHA_Encryptor rsaoaep;  //Fill with OAEP encryption
    int r = rsaoaep.enc(input, output);
    ...
}
