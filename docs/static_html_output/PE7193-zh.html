缺陷描述

构造方法定义混乱，保证一个标准的构造函数

漏洞与风险

这个常规方法的名称与它所定义的类同名，这很可能是一个构造函数。 如果想要将它定义为一个构造函数，那么就删除一个void返回值的声明。 如果您不小心定义了这个方法，并已经意识到这个错误且定义一个合适的构造函数 但是却由于向后兼容问题不能解除这个方法，那么不应该使用这种方法。
