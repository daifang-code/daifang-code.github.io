缺陷描述

布尔同步

漏洞与风险

代码同步使用了装箱的基本类型，例如布尔值。

private static Boolean inited = Boolean.FALSE;
...
synchronized(inited) {
    if (!inited) {
        init();
        inited = Boolean.TRUE;
    }
}
...

由于通常只有两个TRUE和FALSE布尔对象，所以这段代码可能和另一段不相关的代码同步使用了同一个对象，这将可能导致不响应和死锁。

如果需要查看更多信息，请参考CON08-J. Do not synchronize on objects that may be reused
