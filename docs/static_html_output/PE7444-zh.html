缺陷描述

由于重新使用Entry对象，添加entry集合的元素可能会失败

漏洞与风险

允许entrySet()方法返回底层Map的视图，其中在迭代期间重新使用并返回单个Entry对象。从Java 1.6开始，IdentityHashMap和EnumMap实行了上述的做法。迭代这样的Map时，Entry值仅在您到达下一次迭代之前有效。例如，如果您尝试将这样的entrySet传递给addAll方法，那么代码就会出错。
