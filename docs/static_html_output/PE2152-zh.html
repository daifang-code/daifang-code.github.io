禁用__proto__（no-proto）

__proto__ 属性在 ECMAScript 3.1 中已经被弃用，并且不应该在代码中使用。使用 getPrototypeOf 方法替代 __proto__。

缺陷详因

当一个对象被创建，__proto__ 被设置为对象构造方法的原始原型属性。getPrototypeOf是获取原型的首选方法。

错误 代码示例：

/*eslint no-proto: "error"*/

var a = obj.__proto__;

var a = obj["__proto__"];

正确 代码示例：

/*eslint no-proto: "error"*/

var a = Object.getPrototypeOf(obj);

当该情况下该规则可忽略

如果你需要支持老版本的浏览器，你可能会想要关闭此规则，因为 getPrototypeOf 还没有被广泛支持。
