缺陷描述

在装箱的基本类型上同步

漏洞与风险

代码同步使用了一个装箱基本类型常量，例如一个整数。

private static Integer count = 0;
...
synchronized(count) {
    count++;
}
...

由于整数对象可以被缓存和共享，因此这段代码可以与其它不相关的代码同步使用相同的对象，从而导致不响应和死锁。

如果需要查看更多信息，请看参考CON08-J. Do not synchronize on objects that may be reused
