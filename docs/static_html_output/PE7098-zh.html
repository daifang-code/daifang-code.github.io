缺陷描述

Math.max和Math.min的错误组合

漏洞与风险

此代码尝试使用类似Math.min(0,Math.max(100，value))来限制value边界。 但是函数内常量的顺序不正确：本应该是Math.min(100，Math.max(0，value))。 结果，此代码始终生成相同的结果（如果value为NaN，则为NaN）。
