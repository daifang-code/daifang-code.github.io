PE0009: Char I/O Return Error

The return value of certain stdio functions is incorrectly assigned to a char variable instead of an int.

Bug info

BugTypeID
PE0009
Tags
C, CPP
Category
Checkers/CPP
Authors
pinpoint

Description

Functions fgetc(), getc(), getchar(), and istream::get(), which return int values, not char values. Assigning an int value to a char variable truncates the value. If the char variable is unsigned, the loworder 8 bits of the int value are not put into the char variable, which is a modulo operation if the int value is non-negative. If the int value is -1, the low-order 8 bits of the representation of -1 are put into the char variable, which results in a value of 255. If the char variable is signed, an int value of -1 remains -1 in the char variable, but any int value greater than 127 becomes a negative value in the char variable. Using a signed char variable is also unsafe. A stray Ó± character (0xFF) will cause the program to erroneously think that EOF is reached. The C standard requires that such functions be passed an integer in the range of -1 to 255. A signed char variable can potentially have values as low as -128.

Vulnerability and risk

truncate value error

Assigning an int value to a char variable truncates the value.

Likelihood of Exploit

Medium

Potential Mitigations

Assigning the return value of functions fgetc(), getc(), getchar(), and istream::get() to a int value.

Demonstrative Examples

Vulnerable Code Example
int bad() {
  char c;
  c = getc(stdin);
}

In the vulnerable code example, returns an int value to variable c, which is a char value at line 3.

Fixed Code Example
int good() {
  int i;
  i = getc(stdin);
}

In the fixed code example, returns an int value to variable i, which is a int value at line 3.
