缺陷描述

使用不可信的输入数据构建HTTP cookie

漏洞与风险

此代码使用不受信任的HTTP参数构造HTTP Cookie。 如果将此cookie添加到HTTP响应中，则可能出现HTTP响应拆分漏洞。 需要在构造HTTP cookie前先检查HTTP参数中是否有恶意注入代码。查看 http://en.wikipedia.org/wiki/HTTP_response_splitting 获取更多信息。
