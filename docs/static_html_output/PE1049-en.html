PE1049: Incorrect exception throw/catch

Incorrect exception throw/catch

Bug info

Authors

Description

Throughing exception in a destructor is dangerous because the catching of the exception is likely to miss.

Exception points at deallocated memory.

Performance issue. Throwing a copy of the caught exception instead of rethrowing the original exception. Rethrowing an exception with 'throw + varname;' creates an unnecessary copy. To rethrow the caught exception without unnecessary copying or slicing, use a bare throw.

Performance issue. Exception should be caught by reference instead of value to avoid memory copying.

Exception thrown in function declared not to throw exceptions.
