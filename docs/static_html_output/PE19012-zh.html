安全风险

SQL注入形成原因
    SQL语句用户可控，若未对参数进行足够的过滤则会导致SQL注入
风险
    - 用户信息被篡改
    - 攻击者偷取代码和用户数据
    - 业务运营的所有数据被恶意获取
    - 线上代码被非法篡改，并造成为恶意攻击者输送流量或其他利益的影响

修复方案

    - 已不再使用的功能，直接做下线处理
    - 引入安全组件
    - 代码层修复:对用户可控参数进行严格过滤
