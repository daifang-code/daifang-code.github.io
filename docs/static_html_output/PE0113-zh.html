PE0113: 过时的implicit int

过时的implicit int。

缺陷信息

缺陷ID
PE0113
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

在声明函数或变量时，使用了过时的"implicit int"规则。

漏洞与风险

当一个变量未被指定所属类型时，默认的是"int"型，但现代C++已不再支持此规则，可能导致代码难以移植。

漏洞利用威胁

低

缓解与预防

在声明函数或变量时，应指定变量或函数返回值的类型。

演示实例

漏洞代码示例
const a = 0;   // 未声明常量的类型
example_fun()    // 未声明函数返回值的类型
{
  ...
}
漏洞修复代码示例
const int a = 0;   
int example_fun()    
{
  ...
}
