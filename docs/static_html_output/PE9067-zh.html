PE9067: 多余的头文件引用

在C和C++源码中寻找多余的头文件引用。

缺陷信息

缺陷ID
PE9067
标签
C, CPP
目录
Checkers/CPP
作者
pinpoint

缺陷描述

这个检查器通过分析C和C++代码文件来决定是否可以有一些头文件可以被移除而不影响源码的编译。 这个可以带来编译时间的提升，特别是在一些大的C++项目中。移除头文件有时也会暴露开发者都没有意识到的依赖问题。

漏洞与风险

出现多余的头文件可以导致：

  • 降低编译构建速度
  • 导致冗余的重编译
  • 导致无效的Refactoring
  • 使项目的文档不再自包含（即由于多余头文件的存在，你不在能依赖头文件包含关系来推断项目文件依赖）。

漏洞利用威胁

中

缓解与预防

从源码中删除对头文件的引用。

演示实例

漏洞代码示例
#include <iostream>  // Redundant Header Includes

int test() {
    return 1 + 2;
}

在上面的代码中，头文件 <iostream> 的任何符号都没有被用到，所有改头文件是多余的且应该被移除掉。
