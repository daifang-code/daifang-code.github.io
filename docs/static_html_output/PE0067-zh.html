PE0067: 变量赋值后未使用

变量在下一次赋值操作或函数结束之前没有被使用过。

缺陷信息

缺陷ID
PE0067
标签
C, CPP, JAVA ,FIND-BUGS
目录
Checkers/CPP
作者
pinpoint

缺陷描述

变量在下一次赋值操作或函数结束之前没有被使用过，那么这样的赋值或初始化实际上是无效的。这样的编码方式可能会引入逻辑错误。

漏洞与风险

无效代码

变量在下一次赋值操作或函数结束之前没有被使用过，那么这样的赋值或初始化实际上是无效的。

漏洞利用威胁

低

缓解与预防

移除无效的代码。

演示实例

漏洞代码示例
int bad() {
  int x;
    x = 10; // Value stored to x is never read.
    x = 20;
    return x;
}

在上面的代码中，第3行的赋值语句是无效代码。

漏洞代码修复示例
int good() {
  int x;
    x = 20;
    return x;
}

在上面的代码中，移除无效代码。
