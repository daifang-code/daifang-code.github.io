PE0136: Reveal stack traces

Error handling reveals stack traces or other overly informative error messages to users.

Bug info

BugTypeID
PE0136
Tags
JAVA
Category
Checkers/JAVA
Authors
pinpoint

Description

The application serverâ€™s configuration allows detailed error messages, e.g. stack traces, to be returned to users. This potentially exposes sensitive information or underlying flaws such as component versions that are known to be vulnerable.

Vulnerability and risk

Such flaws frequently give attackers unauthorized access to some system data or functionality. Occasionally, such flaws result in a complete system compromise.

Potential Mitigations

Do not reveal stack traces.

Likelihood of Exploit

Critical

Demonstrative Examples

Vulnerable code Example
public class Example {
  try{
    ...
    }catch(Exception e) {
      Logger log = Logger.getLogger("lavasoft");  
      log.log(Level.INFO, "dfsf", e);
    }
    finally{
    }
  }
}

The exception message e is exposed in the log.

Fixed code Example
public class Example {
  try{
    ...
    }catch(Exception e) {
      Logger log = Logger.getLogger("lavasoft");  
      log.log(Level.INFO, "dfsf");
    }
    finally{
    }
  }
}

Remove the exception message e.
