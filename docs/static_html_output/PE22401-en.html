PE22401: E266

Too many leading '#' for block comment

Bug info

BugTypeID
PE22401
Tags
flake8
Category
Code style and efficiency
Authors
pinpoint

There should be only one leading # for a block comment.

Anti-pattern
## Prints hello
print('hello')
Best practice

Only use one #.

# Prints hello
print('hello')
