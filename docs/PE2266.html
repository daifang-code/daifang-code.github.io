no-wrap-func: disallow unnecessary parentheses around function expressions

(removed) This rule was removed in ESLint v1.0 and replaced by the no-extra-parens rule. The "functions" option in the new rule is equivalent to the removed rule.

Although it's possible to wrap functions in parentheses, this can be confusing when the code also contains immediately-invoked function expressions (IIFEs) since parentheses are often used to make this distinction. For example:

1
2
3
4
5
6
7
var foo = (function() {
    // IIFE
}());

var bar = (function() {
    // not an IIFE
});

Rule Details

This rule will raise a warning when it encounters a function expression wrapped in parentheses with no following invoking parentheses.

Example of incorrect code for this rule:

1
var a = (function() {/*...*/});

Examples of correct code for this rule:

1
2
3
var a = function() {/*...*/};

(function() {/*...*/})();
